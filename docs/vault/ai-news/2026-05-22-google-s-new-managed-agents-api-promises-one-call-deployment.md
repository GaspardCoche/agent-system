    # Google's new Managed Agents API promises one-call deployment at the cost of execution-layer control

![Google's new Managed Agents API promises one-call deployment at the cost of execution-layer control](https://venturebeat.com/_next/image?url=https%3A%2F%2Fimages.ctfassets.net%2Fjdtwqhzvc2n1%2F56jT0kiqeEVuPHMFht0VT6%2Ff33bf9f5979e611149822ec03373eb05%2Fcrimedy7_illustration_of_a_supervisor_robot_evaluating_other__35a35697-0dc1-4a73-acea-a033e99f2fac_2.png%3Fw%3D1000%26q%3D100&w=3840&q=50)

    **Source:** [VentureBeat AI](https://venturebeat.com/orchestration/googles-managed-agents-api-promises-one-call-deployment-at-the-cost-of-execution-layer-control) | **Categorie:** Outils & Plateformes | **Importance:** must_read

    Google lance une nouvelle API Managed Agents, conçue pour simplifier le déploiement d'agents IA en réduisant des semaines de travail à un seul appel. Bien que cela accélère considérablement l'intégration des agents, elle implique de céder à Google le contrôle de la couche d'exécution. Pour les entreprises, c'est un compromis entre une rapidité de déploiement inégalée et une certaine dépendance vis-à-vis de l'infrastructure Google, affectant potentiellement la flexibilité et la personnalisation.
